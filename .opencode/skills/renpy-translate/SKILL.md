## What I do

Перевожу визуальные новеллы Ren'Py на целевой язык. Все переводы в этом проекте — в едином формате `translate <lang> strings:` с парами `old`/`new`:

```rpy
translate ru strings:

    old "Original English text."
    new "Перевод на русский."
```

- Перевожу диалоги, нарратив и строки интерфейса — всё через old/new
- Исправляю ошибки согласования (род, число, падеж)
- Проверяю отсутствие пустых переводов
- Проверяю на дубликаты блоков

## Порядок работы

1. Прочитай файл или группу файлов
2. Переведи все строки с пустым/английским `new`
3. Проверь, что нет пропущенных переводов
4. Проверь на дубликаты

## Типичные ошибки

#### 1. Непереведённые строки (new копирует old)

По умолчанию `new` заполнен копией `old`, чтобы игра показывала английский текст, пока перевода нет. Нужно найти строки, где `new` всё ещё равен `old`:

**Проверка:**
```bash
grep -rn '    new "' tl/<lang>/ | grep -v '"$' || true
```
Либо визуально — если в строке `new` и `old` совпадают, строка не переведена.

**Исправление:** Заменить содержимое `new` на русский перевод. Пустых `new ""` в проекте нет.

#### 2. Английские слова, оставленные в `new` при быстром переводе

Самая частая ошибка при пакетном переводе — часть слов остаётся на английском. Особенно часто страдают:
- наречия: `almost`, `especially`, `probably`, `absolutely`, `clearly`, `indeed`, `never`, `quickly`, `slowly`
- глаголы: `feel`/`felt`/`feels`, `faced`, `touched`, `gliding`/`glided`, `twisting`, `pushing`, `wondered`, `spasmed`, `continued`, `resumed`
- прилагательные: `confused`, `surprised`, `distressed`, `intense`, `coherent`
- существительные: `bulge`, `satisfaction`, `routine`, `sensation`, `amused`

**Исправление:** Заменить каждое найденное английское слово на русский аналог.

#### 3. Ошибки согласования

**Плохо:** `new "Персонаж делает что-то, обычно такую делающей"` (ошибка падежа)
**Хорошо:** `new "Персонаж делает что-то, обычно так делающую"`

#### 4. Род глагола при обращении (ты + глагол)

В английском нет различия по роду в глаголах прошедшего времени (`you saw`), но в русском нужно выбирать род адресата. Особенно важно в диалогах, где мужской персонаж обращается к женскому и наоборот.

**Плохо (король — Саре):**
```rpy
    old "Speaking of growing up, have you seen..."
    new "Кстати о взрослении, ты не видел..."
```

**Хорошо (король — Саре, female):**
```rpy
    old "Speaking of growing up, have you seen..."
    new "Кстати о взрослении, ты не видела..."
```

**Проверка:** Все диалоги с `ты` + глагол прошедшего времени должны быть проверены на род адресата. Особенно обращать внимание на:
- мужчина → женщина: глагол должен быть женского рода
- женщина → мужчина: глагол должен быть мужского рода

**Типичные глаголы, требующие проверки:**
`видел/видела`, `сказал/сказала`, `был/была`, `думал/думала`, `знал/знала`, `мог/могла`, `хотел/хотела`, `пошёл/пошла`, `пришёл/пришла`, `стал/стала`, `взял/взяла`, `дал/дала`, `понял/поняла`, `увидел/увидела`, `начал/начала`

#### 5. Дублирующиеся блоки `translate`

Один и тот же `translate <lang> strings:` не должен встречаться дважды в одном файле.

## Единственный формат: `translate <lang> strings:`

```
translate <lang> strings:
    old "Original text"
    new "Translated text"
```

- `old` — оригинальный текст (не трогать)
- `new` — перевод (заполнить/отредактировать)

### Имена персонажей (из `"Character" "text"`)

Когда оригинальный скрипт содержит `"Character" "text"` (имя персонажа в кавычках), экстрактор автоматически разделяет их на две сущности:
- Имя персонажа → `misc_strings.rpy` (character_name)
- Текст реплики → файл сцены (dialogue)

Это значит, что имя персонажа переводится отдельно, в `misc_strings.rpy`. Например:

```rpy
# В misc_strings.rpy:
translate ru strings:
    old "Barkeeper"
    new "Бармен"

# В файле сцены:
translate ru strings:
    old "Basic dances, nothin fancy..."
    new "Базовые танцы, ничего особенного..."
```

### Длинные нарративные строки

Нарратив может быть длинным описанием (сцены, действия, мысли). Переводи свободно, добиваясь естественного звучания:

```rpy
    old "King Orwell stares out onto the courtyard, a stern look on his face. Snow is slowly drifting down."
    new "Король Орвелл смотрит во двор, на его лице строгое выражение. Медленно кружится снег."
```

## Справочно: диалоговые блоки (в этом проекте НЕ ИСПОЛЬЗУЮТСЯ)

В других проектах Ren'Py перевод диалогов может выглядеть так:

```rpy
translate ru start_a1b2c3d4:

    # e "Original dialogue."
    e "Переведённая реплика."
```

Переменная персонажа (`e`, `sarah` и т.д.) **НЕ переводится** — это Python-идентификатор.
Такой формат в этом проекте отсутствует; весь контент идёт через `strings:`.

## Когда перевод готов

- Нет строк, где `new` всё ещё совпадает с `old` (т.е. нет непереведённых)
- Нет дубликатов `translate <lang> strings:`
- Нет слов исходного языка в переводе (проверить скриптом из ошибки №2)
- Нет одиночных английских слов, случайно оставленных в русском тексте
- Имена персонажей из `"Character" "text"` переведены в `misc_strings.rpy`
- Число строк `old` и `new` совпадает
- Глаголы при `ты` согласованы по роду с адресатом (ошибка №4)
- Нет дублирующихся `old` строк в разных файлах (одинаковый `old` в разных файлах → Ren'Py ошибка)
