## What I do

Перевожу визуальные новеллы Ren'Py на целевой язык с соблюдением формата Ren'Py translation:

- Перевожу диалоги и нарратив в script.rpy через translate-блоки
- Перевожу строки интерфейса в screens.rpy и common.rpy через old/new формат
- Исправляю ошибки согласования (род, число, падеж)
- Проверяю отсутствие пустых переводов

## Работа с split-файлами (рекомендуемый workflow)

### Где переводить

Все переводы выполняются в **decomposed чанках**:
```
tl/<lang>/script/split/
├── Arc1/
│   ├── Scene1.rpy
│   └── ...
├── Arc2/
│   └── ...
└── ...
```

### Порядок работы

1. Прочитай файл или группу файлов
2. Найди пустые переводы
3. Переводи блоками (определяй удобный размер сам: 30-100 штук)
4. Проверяй отсутствие пустых после перевода
5. Проверяй на дубликаты блоков

### Формат translate-блока

```rpy
translate <lang> SceneName_hash_suffix:

    # "Original text in source language."
    "Translation in target language."
```

### Типичные ошибки

#### 1. Дублирующиеся блоки

**Проблема:** Один и тот же `translate <lang> SceneName_hash` встречается дважды с разным содержимым.

**Решение:** Найти и удалить дубликаты, оставив только первый (или более полный) вариант.

#### 2. Пустые переводы

**Проверка:**
```bash
grep -rn '    ""' tl/<lang>/script/split/
```

**Исправление:** Заполнить пустой перевод соответствующим текстом.

#### 3. Слова исходного языка в переводе

**Плохо:** `"Текст like this, more text"`
**Хорошо:** `"Текст, подобный этому, ещё текст"`

#### 4. Ошибки согласования

**Плохо:** `"Персонаж делает что-то, обычно такую делающей"` (ошибка падежа)
**Хорошо:** `"Персонаж делает что-то, обычно так делающую"`

## Основные правила

### 1. Всегда переводи на чистом, грамотном языке

Это художественная новелла — перевод должен быть красивым и читаемым.
Следи за согласованием слов (род, число, падеж).
Не переводи дословно, а передавай смысл.
Если фраза звучит неестественно — переформулируй.

### 2. Формат script.rpy (диалоги и нарратив)

Каждый блок имеет уникальный ID. Оригинал ЗАКОММЕНТИРОВАН, перевод — на следующей строке:

```rpy
translate <lang> unique_id:

    # "Original narrative or dialogue."
    "Translation."
```

### 3. Формат screens.rpy / common.rpy (интерфейс)

Формат old/new внутри блока `translate <lang> strings:`:

```rpy
translate <lang> strings:
    old "English string"
    new "Translated string"
```

### 4. Две фразы подряд в кавычках (персонаж + текст)

Когда в оригинале `"Speaker" "Speech"` — это **имя персонажа + его реплика**:

```rpy
translate <lang> unique_id:

    # "Speaker" "Speech"
    "Персонаж" "Речь"
```

**ВНИМАНИЕ:** НЕ используй `new` — просто две строки в кавычках!

## Когда перевод готов

- Игра запускается без ошибок
- Диалоги отображаются на целевом языке
- Нет пустых переводов
- Нет дубликатов блоков
- Нет слов исходного языка в переводе
