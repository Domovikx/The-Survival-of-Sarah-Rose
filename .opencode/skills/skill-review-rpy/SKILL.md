# skill-review-rpy

Проверяю качество перевода одного .rpy файла.

## Критерии проверки

### 1. old-строки не повреждены

Сравни `old` строки с оригиналом (git HEAD). Если `old` изменился — это критическая ошибка.

**Проверка:**

```bash
git diff HEAD -- <путь_к_файлу>
```

Если в diff есть изменения в строках, начинающихся с `old "` — файл повреждён.

### 2. Все строки переведены

Нет строк, где `new` совпадает с `old`.

**Проверка:** Найди все блоки `translate ru strings:` и проверь каждую пару old/new.

### 3. Нет английских слов в new

`new` не должен содержать латинских слов, кроме:

- Имён собственных персонажей (Sarah, Rose, King, Orwell и т.д.)
- Спецтерминов (Ren'Py, Steam, PC, OK)
- Единиц измерения (kg, km, %)

**Проверка:** Найди латинские слова в `new` строках. Каждое слово длиннее 2 символов, состоящее из латиницы, должно быть переведено.

### 4. Синтаксис Ren'Py

- Каждый `translate ru strings:` блок имеет правильные отступы (4 пробела)
- Пары old/new идут последовательно
- Нет пустых `new ""` строк
- Нет дублирующихся блоков `translate ru strings:` внутри файла

### 5. Согласование

- Глаголы при `ты` согласованы по роду с адресатом
- Нет ошибок падежа, числа, рода
- Фразы звучат естественно по-русски

### 6. Полнота перевода

Проверь, что переведены **все** блоки в файле — от первого `translate ru strings:` до последнего. Если в конце файла есть непереведённый блок — укажи его номер/контекст.

### 7. Русская типографика

Проверь пунктуацию в `new` строках:

- **Кавычки**: используй «ёлочки» (Alt+0171 / Alt+0187), а не прямые "лапки". Лапки допускаются только внутри «ёлочек» при необходимости вложенных кавычек.
- **Тире**: длинное тире (—) между частями предложения с пробелами с обеих сторон. Дефис (-) — только внутри слов (кое-как, тёмно-красный). Короткое тире (–) — для диапазонов чисел (5–10).
- **Многоточие**: ровно три точки `...`, в конце предложения четвёртой точкой становится точка предложения (`....`).
- **Пробелы**: нет двойных пробелов, пробел перед длинным тире и после.

**Проверка:** Найди прямые кавычки `"текст"` в `new` строках, одиночные дефисы между словами там, где нужно длинное тире, и неправильное многоточие (2 или 4+ точек).

### 8. Стилистическая естественность (кальки)

Проверь `new` на типичные англо-русские кальки:

- **Пассивный залог**: `было сделано` → `сделал(а)`. Замени пассив на активный, если это звучит естественнее.
- **Причастные обороты**: разбивай длинные английские предложения с `which`/`that` на несколько коротких.
- **Порядок слов**: русский порядок слов свободный — не копируй английский (SVO). Выноси обстоятельства в начало, меняй местами подлежащее и сказуемое для естественного звучания.
- **Буквальные идиомы**: `it's raining cats and dogs` → не `дождь из кошек и собак`, а `льёт как из ведра`. Найди и замени все буквально переведённые идиомы и метафоры.
- **Канцеляризмы**: `осуществлять`, `является`, `представлять собой` — в диалогах звучат неестественно. Замени на живые глаголы.

### 9. Адаптация междометий и звуков

Визуальные новеллы полны эмоциональных выкриков и звуков. Они должны быть русифицированы:

**Проверка:** Найди английские междометия в `new` и замени на русские аналоги:

| Английский                     | Русский                               |
| ------------------------------ | ------------------------------------- |
| Uh... / Umm...                 | Эм... / М-м...                        |
| Ah! / Oh!                      | А! / О! / Ах!                         |
| Hmm... / Hm...                 | Хм...                                 |
| Ugh... / Urgh... / Eugh...     | Тьфу... / Фу... / Угх...              |
| Wow! / Whoa!                   | Ничего себе! / Ого!                   |
| Hey! / Hey!                    | Эй! / Ай!                             |
| Gah! / Argh! / Agh!            | Ай! / Чёрт! / Проклятье!              |
| _sigh_ / _sighs_               | _вздыхает_                            |
| _laugh_ / _chuckle_ / _giggle_ | _смеётся_ / _хихикает_ / _усмехается_ |
| _cough_ / _clears throat_      | _кашляет_ / _прочищает горло_         |
| _gasp_                         | _ахает_ / _затаивает дыхание_         |
| _groan_ / _moan_               | _стонет_                              |
| _whimper_                      | _хнычет_                              |
| _pant_ / _pants_               | _тяжело дышит_                        |

Звукоподражания должны быть русскими, а не транслитом английских.

### 10. Длина строк (fit-проверка)

Русский текст в среднем на 15–30% длиннее английского. Длинные `new` строки могут не поместиться в диалоговое окно и обрезаться движком Ren'Py.

**Проверка:** Если длина `new` более чем в 1.5 раза превышает длину `old` — отметь строку как потенциально проблемную. Предложи вариант сокращения без потери смысла.

### 11. Числа прописью

В русской художественной литературе числа от 1 до 10 принято писать словами, не цифрами. Исключение: даты, игровые механики, меры, статистика, цены.

**Проверка:** Найди цифры 1–10 в нарративе и диалогах (не в интерфейсных строках). Если число написано цифрой, а не словом — отметь строку.

### 12. Консистентность терминов

Один и тот же английский термин должен переводиться одинаково во всех строках файла.

**Проверка:** Найди повторяющиеся `old` и сравни их `new`. Если одинаковый `old` переведён по-разному — отметь как нарушение консистентности. Имена персонажей, титулы (`King`, `Queen`, `Lord`), названия локаций, рас, магических терминов должны быть единообразны.

### 13. Специфика 18+ контента

В игре есть эротические сцены. При проверке таких строк учитывай:

- **Степень откровенности** должна соответствовать оригиналу. Не делай перевод более вульгарным или более целомудренным, чем оригинал.
- **Эвфемизмы**: английские метафоры для интимных терминов заменяй русскими аналогами, а не транслитом или кальками. Если в оригинале используется поэтичное описание — в переводе тоже должна быть поэтика, а не медицинские термины (и наоборот).
- **Повторяющиеся термины** (названия частей тела, действий) — единообразно во всех интимных сценах.
- **Нецензурная лексика**: если в оригинале есть мат/сленг — передавай эквивалентом по силе звучания и стилистической окраске. Если в оригинале мата нет — не добавляй.
- **Согласование рода** в интимных сценах особенно важно: следи, чтобы глаголы при `ты` были согласованы с полами персонажей.

## Формат ответа

Если всё хорошо:

```
approved
```

Если есть проблемы:

```
rejected
Проблемы:
1. Строка 45: не переведено (new == old)
2. Строка 123: английское слово "survival"
3. Строка 78: глагол не согласован, мужчина → женщина: "видел" → "видела"
```

Каждая проблема должна содержать номер строки и конкретное описание, чтобы переводчик мог точно исправить.
